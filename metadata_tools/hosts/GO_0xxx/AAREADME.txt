                RMS NODE METADATA FOR PDS3 VOLUMES GO_0002-0023
                -----------------------------------------------

This is a collection of metadata maintained by the Ring-Moon Systems Node and
used in the OPUS search engine:
    https://opus.pds-rings.seti.org

Each subdirectory contains index files for the volumes of the same name. The
final subdirectory, GO_0999, contains cumulative indices for all volumes.

*_index.tab                 A copy of each index file as originally provided by
                            the team, with updates and corrections. See details
                            below.
*_index.lbl                 PDS3 label for the above.

*_supplemental_index.tab    An index of supplemental information not found in
                            the original index. This content has been scraped
                            or derived from the product labels.
*_supplemental_index.lbl    PDS3 label for the above.

*_ring_summary.tab          Index of viewing geometry for Jupiter's ring system,
                            provided for volumes GO_0016-GO_0023.
*_ring_summary.lbl          PDS3 label for the above.

*_body_summary.tab          Index of viewing geometry for all relevant planets
                            and moons, provided for volumes GO_0002- GO_0023.
*_body_summary.lbl          PDS3 label for the above.

*_sky_summary.tab           Index of sky viewing geometry for all volumes.
*_sky_summary.lbl           PDS3 label for the above.

GO_0016_sl9_index.tab       Summary information about each of the image files on
                            volume GO_0016.
GO_0016_sl9_index.lbl       PDS3 label for the above.


RELEASE NOTES
-------------

Version 5.0: February 26, 2025

This major release involves the reorganization of the geometry index files for
the Galileo mission, as well as the addition of new geometric quantities.

All body geometry is now tabulated in a single body table. The following
geometric quantities have been added to the ring geometry index files:
  LONGITUDAL_RESOLUTION
  RADIUS_IN_PIXELS
  CENTER_X_COORDINATE
  CENTER_Y_COORDINATE
  EDGE_ON_RING_LONGITUDE_WRT_NODE
  RING_LONGITUDE_WRT_NODE
  SURFACE_INTERCEPT_TIME

The following geometric quantities have been added to the body geometry index
files:
  RADIUS_IN_PIXELS
  CENTER_X_COORDINATE
  CENTER_Y_COORDINATE
  NORTH_POLE_CLOCK_ANGLE
  NORTH_POLE_POSITION_ANGLE
  LIMB_ALTITUDE
  LIMB_CLOCK_ANGLE
  SURFACE_INTERCEPT_TIME


Version 4.0: August 20, 2024

This major release includes newly-generated geometric metadata for the entire
Galileo mission, as well as new supplemental indices of geometry missing from
the original index files.


Version 3.0: February 26, 2022

The latest release includes all volumes of Galileo SSI cruise data, spanning
volumes GO_0002-0016, in addition to the Jupiter tour data in volumes
GO_0017-0023, which were provided in the previous release of metadata. The
calibration files found on volume GO_0001 are not included.

In this release, index file columns have been modified to ensure that all
indices have the same format. Four unused columns (MEAN_RADIANCE,
MEAN_REFLECTANCE, RADIANCE_SCALING_FACTOR, and REFLECTANCE_SCALING_FACTOR) have
been removed.

The first rows of GO_0002_index.tab contained impossible time tags. These have
been replaced with "UNK".

The new index file GO_0016_sl9_index.tab contains information unique to the SL9
observations. Each SL9 image file contains multiple observations and appears in
multiple rows of GO_0016_index.tab, each row having a different IMAGE_ID. In
GO_0016_sl9_index.tab, there is one row per image file rather than one row per
IMAGE_ID, and the row summarizes the information about the image.


Version 2.0: October 3, 2018

The original volumes GO_0017-0023 were subject to an early CD-ROM requirement
that file names could only have eight characters, plus a three-character
extension. As a result, Galileo SSI file names were split in the middle, with
the first seven characters used as a directory name and the remaining
characters used as the name of the file inside that directory. For example,
file C0520792800R.IMG was archived as C052079/2800R.IMG

In this set of revised metadata, all data file paths have been restored to
their original names.
